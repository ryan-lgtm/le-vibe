## `LE_VIBE_EDITOR` (launcher ↔ IDE binary)

The Python stack (`lvibe`, `le-vibe` wrappers) resolves the desktop editor in this order (see `le-vibe/le_vibe/launcher.py`):

1. **`LE_VIBE_EDITOR`** if set in the environment (absolute path or executable name on `PATH`).
2. Else **`/usr/lib/le-vibe/bin/codium`** if that file exists and is executable (packaged **`le-vibe-ide`** **`.deb`** — **`packaging/debian-le-vibe-ide/`**; **PRODUCT_SPEC** §7.3: internal path only; public **`PATH`** CLI remains **`lvibe`**).
3. Else **`/usr/bin/codium`** if that file exists and is executable.
4. Else the bare command **`codium`** (must be on `PATH`).

**Persisting `LE_VIBE_EDITOR` (14.g):** set a login/session default with **`export LE_VIBE_EDITOR=/absolute/path/to/codium`** in **`~/.profile`**, **`~/.bashrc`**, or a **`systemd` user environment file** under **`~/.config/environment.d/`** (`*.conf`) so GUI sessions and terminals inherit it before **`lvibe`**. Stack defaults and **`le-vibe-ide`** packaging are summarized in **`debian/le-vibe.README.Debian`** and **[`BUILD.md`](BUILD.md)** *14.g*.

**Packaging:** the **`le-vibe`** `.deb` uses **`Recommends: codium`** and **`Suggests: le-vibe-ide`** in `debian/control` so a typical install gets an editor without extra flags; the optional **`le-vibe-ide`** package (sibling Debian source under **`packaging/debian-le-vibe-ide/`**) installs the IDE tree and **`/usr/lib/le-vibe/bin/codium`**. See **[`BUILD.md`](BUILD.md)** *Default `LE_VIBE_EDITOR` / packaging story when the IDE ships* (**14.g**). Until that package is built, set **`LE_VIBE_EDITOR`** to your **`VSCode-linux-*/bin/codium`** build path, symlink, or **`/usr/bin/codium`**, and keep CI smoke aligned.

**Production `.deb` pair (STEP 14 / §7.3):** With **`VSCode-linux-*`** present, from the repository root **`./packaging/scripts/build-le-vibe-debs.sh --with-ide`** produces **`le-vibe_*_all.deb`** + **`le-vibe-ide_*_amd64.deb`** — **[`docs/PM_DEB_BUILD_ITERATION.md`](../docs/PM_DEB_BUILD_ITERATION.md)** (*Success output (`--with-ide`)* — **Full-product install** on stdout); install both **`.deb`** files — **[`packaging/debian-le-vibe-ide/README.md`](../packaging/debian-le-vibe-ide/README.md)** (*Install both packages*); attach both and **`SHA256SUMS`** when publishing — **[`docs/apt-repo-releases.md`](../docs/apt-repo-releases.md)** (*IDE package*, *Maintainer build output*); default **`ci.yml`** **`le-vibe-deb`** is stack-only — **[`docs/PM_STAGE_MAP.md`](../docs/PM_STAGE_MAP.md)** (*H1 vs §7.3 .deb bundles*); **[`docs/PRODUCT_SPEC.md`](../docs/PRODUCT_SPEC.md)** § *Prioritization* — **Release bundles (H1 / STEP 8 vs STEP 14 / §7.3)**.

**CI:** [`.github/workflows/build-le-vibe-ide.yml`](../.github/workflows/build-le-vibe-ide.yml) runs on `workflow_dispatch`, **`workflow_call`**, **`pull_request`** (paths under **`editor/`**, IDE workflows, **`ci-editor-gate`**, **`ci-editor-nvmrc-sync`**, **`ci-vscodium-bash-syntax`**), and tags `ide-v*`. [`.github/workflows/build-linux.yml`](../.github/workflows/build-linux.yml) is a **`build-linux`** label that dispatches the same job (manual only). When upstream is missing, the job **passes** with a skip notice; with the VSCodium submodule, smoke includes **`bash -n`** via [`ci-vscodium-bash-syntax.sh`](../packaging/scripts/ci-vscodium-bash-syntax.sh) and **`editor/.nvmrc`** parity via [`ci-editor-nvmrc-sync.sh`](../packaging/scripts/ci-editor-nvmrc-sync.sh). The **default `linux` job** uploads pre-binary artifacts (`ide-ci-metadata.txt` — layout/ref/sha/run_id plus **`le_vibe_editor_docs=editor/README.md`** for **`LE_VIBE_EDITOR`**; pinned **`vscode-upstream-stable.json`** when the submodule is present) — **`upload-artifact`** sets **retention-days** (bounded pre-binary store; see workflow YAML). Optional **`linux_compile`** + **`vscodium-linux-build.tar.gz`** is **not** the default on **`pull_request`** (see **Full Linux compile** below — **14.e / 14.f**). A future installable **Lé Vibe** editor **`.deb`/bundle** from this tree remains **H6** — **`docs/vscodium-fork-le-vibe.md`**. The workflow sets **`permissions:`** **`contents: read`**, **`actions: write`** (checkout + artifact upload). On **GitHub Actions**, the run **Summary** tab echoes the same **`LE_VIBE_EDITOR`** pointer (**Pre-binary artifact** line) for quick verification. **Local check:** [`./editor/smoke.sh`](smoke.sh) from the repo root (wraps **[`packaging/scripts/ci-editor-gate.sh`](../packaging/scripts/ci-editor-gate.sh)**; also run via **[`packaging/scripts/ci-smoke.sh`](../packaging/scripts/ci-smoke.sh)**).

**Full Linux compile (14.e / 14.f):** optional workflow job **`linux_compile`** runs **[`ci-vscodium-bash-syntax.sh`](../packaging/scripts/ci-vscodium-bash-syntax.sh)** + **[`ci-editor-nvmrc-sync.sh`](../packaging/scripts/ci-editor-nvmrc-sync.sh)** first (fail fast — same gates as **`ci-editor-gate`**), then **[`packaging/scripts/ci-vscodium-linux-dev-build.sh`](../packaging/scripts/ci-vscodium-linux-dev-build.sh)** → **`editor/vscodium/dev/build.sh`** (**`ci-vscodium-linux-dev-build.sh`** enforces **`node --version`** vs **`editor/.nvmrc`** before **`dev/build.sh`** — **14.a** / **14.e**; **`LEVIBE_SKIP_NODE_VERSION_CHECK`** — see **[`BUILD.md`](BUILD.md)** *Compile wrapper vs Node*), when you enable **`vscodium_linux_compile`** (**`workflow_dispatch`**), pass the same flag via **`workflow_call`** (e.g. **[`build-linux.yml`](../.github/workflows/build-linux.yml)**), or push tag **`ide-v*`** — not the default on **`pull_request`**. The job sets **`NODE_OPTIONS=--max-old-space-size=8192`** (same heap cap as **`editor/vscodium/dev/build.sh`**) and installs a bounded apt toolchain — full list + failure modes — **[`BUILD.md`](BUILD.md)** *CI* / *When full compile fails* (**14.e**). On success, the workflow uploads **`vscodium-linux-build.tar.gz`** as a **`le-vibe-vscodium-linux-<run_id>`** artifact; use **[`print-ci-tarball-codium-path.sh`](print-ci-tarball-codium-path.sh)** to resolve **`bin/codium`** for **`LE_VIBE_EDITOR`**. Lé Vibe–**branded** output still depends on **`editor/le-vibe-overrides/`** + **`PRODUCT_SPEC`** §7.2 — see **[`BUILD.md`](BUILD.md)** and **`docs/vscodium-fork-le-vibe.md`**.
